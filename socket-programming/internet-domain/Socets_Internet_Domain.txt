Intenet Domain socket
______________________

Internet domain stram sockets are implemented on top of TCP.

Internet domain datagram sockets are implemented on top of UDP. 

UDP sockets are not-datagrams may be lost, duplicated or arrive in a different order that in which
they were sent.

Sending on UNIX domain datagram  socket will block if the queue of data for receiving socket is full.
By constrast, with UDP, if the incoming datagram would overflow the receiver's queue, then datagram is
silently dropped.

NWtwork Byte Oeder:

Big-Endian byte order: 
                        N       N+1     N+2     N+3
                        3                        0
                        MSB                     LSB

Little-Endian byte order:
                        N       N+1     N+2     N+3
                        0                        3
                        LSB                     MSB        

All devices on the network should follow same byte order.
Standard 'network byte order' happens to be 'big-endian'

Due to differences in the data representation, application that exchange between hetrogeneous system
over the network must adopt some common convention for encoding that data.
The sender must encode data according to this convention, while receiver decodes following the 
same convention.

Marshalling: The process of putting data into a standard format for transmission across a network
is reffered to as Marshalling.
Various Marshalling Standards: 
    XDR             External Data representation
    ASN.1-BER       Abstract Syntax Notation 1
    COBRA
    XML

DNS: (Domain Name Syatem)
getaddinfo() --> obtains IP addresses corresponding to a hostname.
getnameinfo() --> Obtain hostname from IP address

DNS is used to maintain mappings between hostnames and IP addresses.
Before DNS , hostname and IP addresses were defined in a manually maintained local file /etc/hosts.

gethostbyname() --> getaddinfo() --> obtains IP address by searching in this file.
